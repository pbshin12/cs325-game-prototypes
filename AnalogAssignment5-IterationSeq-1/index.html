  • Technical Merit: What is interesting about your game technology? / Why I do not like Phaser

    This game is made using the Unity game engine. There are several reasons why I chose not to use Phaser 3. The first being that it is extremely difficult to see where I am placing my sprites and game objects. Having to figure out where to place objects through trial and error wastes a lot of my time whereas in Unity I can simply click and drag and know exactly where it is placed. Second, I was able to easily create groups without having to manually guess and check where to place objects within said group. This is especially apparent when it comes to hitboxes. Thirdly, I felt creatively restricted when using Phaser and I often do not know what I'm looking for because I can't find what I needed. What's frustrating is that many of the examples are outdated and I often spend hours searching for something end up empty-handed. In Unity, there are thousands of Youtube videos on nearly everything and I was able to figure out what I needed fairly quickly. Also, Unity's documentation is much easier to read and follow than Phaser's, they provide examples as to how it should look like. Fourth, I had a much easier time managing colliders and triggers in Unity since Unity has built-in functions that handle that. Phaser does have collision detection, but it is often very confusing and I often lose track of the many objects that interact with each other. Whereas in Unity, I can generalize a specific group of objects by either referring to their tag, or even their name, which is very convenient. Lastly, I was a bit frustrated with the Update() function in Phaser 3, especially when it comes to basic controls. There was a consensus amongst several of my classmates where they had encountered a strange bug where their character will be stuck moving in one direction. There were also complaints of strange interactions with different objects whenever they try to implement physics. The fix in Unity would be to use FixedUpdate(), but I had an extremely difficult time finding the equivalent in Phaser. The same can be said regarding DeltaTime, which enables me to record the relative time for a given event.

  • Prototype Postmortem: What did you learn from this prototype? What was the easiest or hardest part of making it?

    The hardest for me was bringing all the objects together and provide me the correct data when a player collides on a station. I wanted each station to charge up when a player stands on it, but it was extremely difficult to find methods to directly take an overlap function inside of an object. So I had to hardcode each station's characteristics. Otherwise, I learned a lot about developing on different scenes and scene transitions.

  • Mandated Variety: State which input, randomness, genre, and play style you used in this prototype.

    The input for this game is mostly direct and uses arrow keys to move. The game is mostly deterministic. The play style is in between explorer and collector.
